

그냥 궁금.
localStorage.getItem('어쩌구')를 직접 사용할때는 변경값이 로컬스에서 실시간으로 바로바로 읽혀지는데

로컬스를 읽는 상태변수를
const [변수,set변수] = useState( localStorage.getItem('어쩌구') );
로 정의하고, 프로바이더로 전달받은 Con.변수를 쓸때는

 set변수(변경값)로 변경해도  그 바로 아래서 변수를 읽으면 아직 값변경이 일어나지 않아서 
 랜더링되는 최상단에 코딩하지 않는 이상, 코딩중간에는 변경값을 바로바로 못씀 (리랜더링후 상단에 변경값찍힘 )


 그렇다는것은 
 const [변수,set변수] = useState( localStorage.getItem('어쩌구') ); 로 정의하는 이유가 뭔지 궁금함.
 그냥   localStorage.getItem('어쩌구')로 늘 불러들여서 값을 확인하면 항상 새롭게 읽어들일수있는데

 set변수(변경값) 으로 변경해야 하는 이유가 있는지? 

 ->> set은 랜더링시를 위한 데이터이므로
 중간에   localStorage.getItem('어쩌구') 이 값이 필요하다면 
 변수 =  localStorage.getItem('어쩌구') 이렇게 집어넣어서
 값을 끌어가면서 코드중간중간  사용하고

 set변수('어쩌구') 할때는 하단쯤에서 명시적으로 안헷갈리게 사용하는게 좋다.
 왜냐면 set해봤자 맨위 아니면 변경된 값을 사용할수 없으므로!
